## Part 6: Quick-Start Path for AWS Deployment

If you want to get infrastructure running TODAY, here's the 4-hour path:

```bash
# 1. Set AWS credentials
export AWS_ACCESS_KEY_ID=...
export AWS_SECRET_ACCESS_KEY=...
export AWS_DEFAULT_REGION=us-east-1

# 2. Initialize bootstrap (creates state bucket)
cd iac/bootstrap
terraform init
terraform apply -auto-approve

# 3. Create dev environment
mkdir iac/environments/dev
cp iac/environments/lab/main.tf iac/environments/dev/
cp iac/environments/lab/terraform.tfvars iac/environments/dev/
# Edit dev/terraform.tfvars to set environment = "dev"

# 4. Deploy dev infrastructure
cd iac/environments/dev
terraform init -backend-config="bucket=proops2026-tfstate-quyetnc"
terraform apply

# 5. Configure kubectl
aws eks update-kubeconfig --name helpdesk-dev-cluster --region us-east-1

# 6. Deploy K8s manifests
kubectl apply -f <(kubectl kustomize k8s/overlays/dev/)

# 7. Verify
kubectl get pods
kubectl get svc
```
